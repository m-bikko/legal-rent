# Ключевые решения Web MVP (2026-07-04)

Зафиксированы с владельцем проекта перед стартом:

1. **Web-first, мобильное приложение позже.** Монорепо: `packages/core`/`i18n` переиспользуются будущим Expo-приложением; UI пишется заново. Причина: SEO-критичность веба для маркетплейса + скорость проверки гипотезы.
2. **Supabase как бэкенд** (проект `xmkfklhubdbtsocfwloh`). Supabase MCP окружения смотрит в другой проект — работать через `pnpm db:migrate`.
3. **OTP — dev-заглушка.** Telegram отправка (бот требует /start от пользователя, Gateway платный) отложена; интерфейс `OtpSender` изолирует замену.
4. **Три языка (ru/kk/en) с первого дня** — тест на идентичность ключей словарей.
5. **Данные только через сервер** (service-role + httpOnly JWT), RLS deny-all. Простейший безопасный контур; публичное чтение ленты откроем политиками при необходимости.
6. **Договор — кнопка, не ЭЦП.** Юридически значимое подписание (eGov QR / NCALayer) — этап 2+.
7. **Модерация верификации — вручную в БД.** Админ-панель вне скоупа MVP.
8. **2ГИС — карточка-ссылка**, не embed (нет публичного ключа виджета).
