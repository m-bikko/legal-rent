# Rental Agreement

Упрощённый договор MVP: юридически значимого подписания (ЭЦП/eGov) пока нет — обе стороны жмут кнопку «Подписать» (см. этап 1 ТЗ, [[tz-summary]]).

## Жизненный цикл

1. Владелец свободного объекта привязывает арендатора **по номеру телефона** (арендатор должен быть зарегистрирован — иначе `tenant_not_found`; себя привязать нельзя — `cannot_rent_own`).
2. Создаётся `draft`. Unique-индекс `one_active_agreement_per_property` гарантирует максимум один draft/active-договор на объект (защита от гонок).
3. Каждая сторона подписывает своё поле (`landlord_signed_at` / `tenant_signed_at`). Арендодатель — со страницы объекта, арендатор — из профиля («Мои договоры»).
4. Обе подписи → status `active`, объект → `rented` ([[property]]).
5. «Завершить аренду» (только арендодатель) → `ended`, объект → `free`. Работает и для отмены draft.

## График платежей (с 2026-07-05)

При привязке арендодатель выбирает дату начала оплаты и количество периодов (лимиты: 60 мес / 366 дней / 720 часов). Генерируются installments (`payment_installments`): платёж i покрывает `[start+i·unit, start+(i+1)·unit)`, дедлайн — начало периода (предоплата). Месяцы — календарные с клампом дня (31 янв → 28 фев), `addPeriod` в core.

Статусы (`installmentStatus`, чистая функция в core): `paid` / `upcoming` / `warning25` (<25% окна) / `critical10` (<10%) / `overdue`. Окно оплаты — один период до дедлайна (для первого платежа — от создания графика), поэтому пороги автоматически масштабируются: месяц ≈ 7.5 дн / 3 дн, день ≈ 6 ч / 2.4 ч, час ≈ 15 мин / 6 мин.

Отметку «оплачено» ставит/снимает только арендодатель (`POST /api/agreements/{id}/installments/{seq}`). Арендатор видит график в профиле (read-only). Реальные платежи через эквайринг — этап 2+; тогда отметка станет автоматической по вебхуку.

## Будущее (по ТЗ)

Подписание через eGov Mobile QR (физлица) и NCALayer (юрлица), Audit Trail (время, IP, способ подписания).
