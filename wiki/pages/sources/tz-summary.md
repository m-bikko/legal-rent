# ТЗ платформы аренды — саммари

Полный документ: `TZ_platforma_arenda_kz_v1.1.md` (91 КБ). Три продукта в одном брифе: маркетплейс объявлений, юридический сервис (ЭДО + налоги), агрегатор бытовых услуг.

## Этапность (раздел 9 + 10.8)

- **Этап 1 (MVP)** ← реализуется сейчас: маркетплейс, KYC с выбором налогового статуса, упрощённый договор, налоговый «подсказчик». Текущий Web MVP покрывает маркетплейс + KYC + договор-кнопку.
- **Этап 2**: коммуналка через агрегатора (Kassa24/Wooppay), escrow-депозит, скоринг, PRO-подписка
- **Этап 3**: налоговый агент (сценарий Б), маркетплейс услуг, страховки, посуточная аренда
- **Этап 4**: прямые ЕРЦ-интеграции, корпоративный сегмент, white-label

## Ключевые интеграции и их риски

- **NCALayer/eGov Mobile** — подписание; NCALayer требует локального ПО (высокое UX-трение)
- **E-Salyq / КГД** — публичного API не подтверждено (риск №2)
- **Реестр недвижимости** — нет открытого API; MVP: документ + ручная модерация (риск №6)
- **ЖКУ** — реформа ЕРЦ с 20.09.2025: целевая интеграция с региональными ЕРЦ вместо десятков поставщиков (раздел 10.1)
- **Платежи** — только через лицензированного эквайрера со сплитованием ([[tax-logic]])

## Продуктовые концепции (10.4)

Цифровой паспорт объекта, единый счёт месяца, rent-паспорт арендатора, статус налогового агента как конкурентный ров.
