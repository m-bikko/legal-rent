# Налоговая логика (РК, с 01.01.2026)

Ключевая регуляторная база проекта; в MVP заложена только частично (типы аккаунтов), полный налоговый модуль — следующие релизы.

## Режим «Самозанятый» (с 2026)

- Только физлица без наёмных работников; лимит ~300 МРП/мес, 3600 МРП/год
- Учёт — обязательное приложение E-Salyq Business, чек в момент дохода
- **Критично**: разрешена ТОЛЬКО жилая недвижимость (ОКЭД 68201). Коммерческая (68202) — запрещена: нужен ИП/ОУР

**Следствие для продукта** (риск №1 ТЗ): при коммерческом типе объекта нельзя предлагать физлицу режим самозанятого. В MVP частично закрыто структурой аккаунтов ([[verification]]); жёсткая блокировка «самозанятый + коммерческий объект» — TODO следующего релиза налогового модуля.

## Два сценария налоговой интеграции (раздел 4.5 ТЗ)

- **А. «Подсказчик»** (выбран для MVP-этапов): платформа определяет режим и ведёт в E-Salyq Business; ответственности налогового агента нет.
- **Б. «Налоговый агент»** (стратегическая цель): регистрация оператором интернет-платформы, удержание ИПН ~1% + соцплатежи (совокупно ~4%). Требует юридической аккредитации ДО запуска.

## Платёжное ограничение (раздел 10.2 ТЗ)

Платформа не должна касаться денег пользователей: приём в пользу третьих лиц = платёжная услуга (лицензия НБ РК, капитал от 150 млн ₸). Целевая схема — сплитование через лицензированного эквайрера (Freedom Pay / Kaspi / ioka).
