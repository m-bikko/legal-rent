Все начинается с страницы логин/регистрации. Идет выбор "Я хочу арендовать", "Я арендадатель"
------
Арендадатель при регистрации: 
Регистрация: Выбирается через switch "Самозанятый", "ИП/ТОО"
Самозанятый -> "ФИО", "Город", "Номер"(адаптируй под кз номера)
Организация -> "ИИН/БИН", "Название ИП/ТОО", "Город", "Номер"(адаптируй под кз номера)
В обоих сценариях приходит otp по телеге
------

------
Тот кто хочет арендавать при регистрации:
Физ. лицо -> ФИО, город, номер(адаптируй под кз номера)
Организация -> ИП/ТОО -> "ИИН/БИН", "Название ИП/ТОО", "Город", "Номер"(адаптируй под кз номера)
------

Логин: Инпут номера и отправить код, после приходит otp по телеграму

Если вход был в качестве того, кто хочет арендавать, то снизу будет menu-bar в котором "Объявления", "Сохраненные", "Профиль"

Раздел Объявления:
Список объявлении с меняющиемися картинами если их больше одной с отображением цены аренды, фильтра, отображение выбранного города(может быть весь казахстан). можно лайкать
Можно зайти в детали квартиры/дома. Там отображаются детали "Номер для связи"(ватсап, позвонить), "Описание", "Цена", "Фотографии", "показ точки на карте 2гис(preview)". можно лайкать

Раздел Сохраненные:
Список объявлении с пометкой лайкнутые

Раздел Профиль:
Теперь у аккаунтов есть статус верифицирован и не верифицирован.(Изначально не верифицирован)

У сущьности здание есть следующие атрибуты:
- Тип здания(Создай разные типы по типу кв, дом, дача, офис и тд)
- Адрес
- Ссылка на 2GIS
- Цена
- Срок аренды(час/день/мес)
- Фотографии
- Описание
- Номера для связи
- Номера ватсапа
- Статус

Разница между арендателем и арендадателем лишь в том что арендадатель может опубликовать квартиру в профиле. Но так же чтобы арендадатель мог опуликовать ему нужно получить пометку "Верифицирован". В зависимости от ИП/ТОО или Самозанятый пусть будут запрошены данные. Пока для ИП/ТОО пусть запросится файл уведомление о начале деятельности, ИИН/БИН организации(Автоматом подставляются с данных при регистра), название организации(Автоматом подставляются с данных при регистрации), Юр адрес. Для Самозанятых ИИН удостака, ФИО(Автоматом подставляются с данных при регистрации), номер удостака, срок удостака, адрес проживания

А для того кто арендует если захочет стать арендадателем будет в профиле кнопка который запрашивает желание стать арендадателем. При нажатии кнопки автоматический идет уже спрос данных для верификации. Физ. лицо отправится на Самозанятого, а организация на организацию. Пусть в странице верификации будет указано какой тип верификации сдается

Пусть у арендадателся в списке его квартир появится пометка В аренде/Свободен/В архиве. Нам важно чтобы мы могли привязать аккаунт через номер к зданию если ее сдают. Там формируется договор который должны подписать оба(пока пусть просто кнопка подписать). После подписания здание становится в аренде
