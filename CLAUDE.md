# RentLegal KZ — инструкции для Claude

Платформа легальной аренды недвижимости в Казахстане. Текущий этап — Web MVP (см. `docs/superpowers/specs/2026-07-04-web-mvp-design.md`, план — `docs/superpowers/plans/2026-07-04-web-mvp.md`).

## Команды

```bash
pnpm dev          # dev-серверы (web на :3000; порт 3000 бывает занят — используй 3100)
pnpm test         # юнит-тесты core + i18n
pnpm build        # production build
pnpm db:migrate   # миграции db/migrations → Supabase (через POSTGRES_URL_NON_POOLING)
```

## Важные особенности

- **Supabase MCP подключён к ЧУЖОМУ проекту** — не использовать его для этого репо. Правильный проект — в `.env` (`xmkfklhubdbtsocfwloh`); работать через `pnpm db:migrate` или прямое pg-подключение.
- `.env` лежит в корне; в `apps/web/.env` — симлинк на него.
- `@rentlegal/core/otp` — отдельный entrypoint (node:crypto): не реэкспортировать из barrel, не импортировать в клиентские компоненты.
- Данные — только через Route Handlers с service-role; браузер к Supabase не ходит. RLS deny-all.
- i18n: три языка ru/kk/en, ключи словарей обязаны совпадать (есть тест). Новые UI-строки — сразу во все три словаря.
- UI: ant-design v5 + Tailwind, иконки lucide-react, без эмодзи в интерфейсе.
- Статус `rented` у объекта устанавливается только через договор — не менять напрямую.

## Wiki (База знаний)

В проекте есть LLM-поддерживаемая база знаний в `wiki/`.

**При любых вопросах о проекте:**
1. Сначала прочитай `wiki/index.md` — найди релевантные страницы.
2. Прочитай нужные страницы из `wiki/pages/`.
3. Отвечай, опираясь на вики. Если информации нет — ищи в коде и обновляй вики.

**При изменениях в проекте:**
- Если изменение затрагивает архитектуру, модели данных, интеграции или важные концепции — обнови соответствующие страницы в `wiki/pages/`.
- Обнови `wiki/index.md`, если создана новая страница.
- Добавь запись в `wiki/log.md` (дата, тип операции, затронутые страницы).

**Схема вики:** `wiki/CLAUDE.md`
